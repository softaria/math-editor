setupDefaultErrorDecoration

  • setupDefaultErrorDecoration(editor: MathEditor, descriptionsElement: Element, customize?: { descriptionElementProvider?: () => HTMLElement; frameRenderCallback?: (frame: HTMLElement) => void; noErrorsHtml?: string; undefinedStateHtml?: string }): Function
    • Defined in premium-edition/api/util/error-highlight.ts:16

    Returns validation callback implementing (demonstrative) reporting of semantic errors:

    1) checks for errors in given editor, 2) if none found - renders no-errors HTML as descriptions, 3) otherwise puts 'error-frame' class on appropriate nodes, 4) renders error descriptions with 'error-description' class for each, 5) connects frames with descriptions by putting 'error-highlight' class on corresponding description when hovering frame.

    Callback is suitable to be called either on a validation button pressed by user (eg. index.html) or programmatically (eg. demo.html).

    Make sure to have styles for '.error-frame', '.error-description' and '.error-description.error-highlight'.
